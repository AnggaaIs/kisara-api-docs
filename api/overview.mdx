<Card title="Rekomendasi Implementasi" icon="sparkles" horizontal>

## Authentication

Endpoint privat menggunakan Bearer token JWT.

<CodeGroup>

```http HTTP Header
Authorization: Bearer <access_token>
```

```bash cURL Example
curl -X GET "https://api.kisara.my.id/endpoint" \
  -H "Authorization: Bearer <access_token>"
```

</CodeGroup>

<Warning>
  Request tanpa token valid akan ditolak. Simpan token di tempat aman dan hindari expose di sisi client publik.
</Warning>

## Rate Limiting

API menerapkan rate limit global dan per-route untuk menjaga stabilitas layanan.

<Steps>
  <Step title="Deteksi Kegagalan Sementara">
    Jika limit terlewati, request akan ditolak sementara oleh server.
  </Step>
  <Step title="Lakukan Retry Bertahap">
    Terapkan exponential backoff untuk menurunkan risiko gagal berulang.
  </Step>
  <Step title="Kurangi Burst Request">
    Gunakan batching atau queue untuk endpoint berat dan operasi paralel.
  </Step>
</Steps>

## Response Format

Sebagian besar endpoint mengembalikan struktur respons berikut:

```json
{
  "status_code": 200,
  "message": "OK",
  "data": {},
  "metadata": {
    "method": "GET",
    "url": "/endpoint",
    "execution_time_ms": 12,
    "ip": "127.0.0.1",
    "timestamp": "2026-04-14T12:00:00.000Z"
  }
}
```

<Info>
  Perlakukan response di atas sebagai kontrak utama. Selalu validasi status_code dan struktur metadata di sisi client.
</Info>

## Praktik Terbaik

<CardGroup cols={2}>
  <Card title="Selalu Cek status_code" icon="circle-check">
    Jangan hanya mengandalkan HTTP status. Gunakan status_code dan message untuk validasi hasil.
  </Card>
  <Card title="Simpan metadata" icon="database">
    Catat metadata.timestamp dan metadata.execution_time_ms untuk audit dan debugging.
  </Card>
  <Card title="Gunakan Timeout" icon="timer">
    Set timeout request di client agar aplikasi tidak menunggu terlalu lama saat jaringan bermasalah.
  </Card>
  <Card title="Gunakan Idempotency" icon="repeat">
    Untuk operasi tertentu, hindari duplikasi request agar tidak menimbulkan efek samping.
  </Card>
</CardGroup>
